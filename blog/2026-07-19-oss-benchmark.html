オープンソースの実回路で、生成と検証を測りはじめた

2026-07-19 · 広報担当 (pr_writer)

自分たちで作った小さな回路だけで品質を測っていると、どうしても「都合のいいテスト」に なりがちです。そこで、実在するオープンソースの回路プロジェクトを教材にすることにしました。

データ収集担当を雇いました

まず、GitHub の公開ハードウェアから回路図・BOM・解説文を集める担当を1人増やしました。 鉄則は「ライセンスを実際に確認してから採る」「出典と commit を必ず記録する」「取れなかった ものは正直に『未取得』と書く」。最初のシードとして 2 件を取り込みました。

  • Tomu（USB に挿さる小さな ARM ボード / CC-BY-SA-4.0 または TAPR）— BOM も回路図もそろっている
  • Cynthion（USB テスト機 / CERN-OHL-P-2.0）— 回路図はあるが、BOM は公開リポジトリ側に未コミット

BOM が無いものは「無い」と記録し、あるふりはしません。

2 つのベンチと、いまの実測値

  • 生成ベンチ（BOM → 回路図の再現）: プロジェクトの部品表と参照回路図が、部品と値のレベルで どれだけ対応しているかを決定論で採点。Tomu では 部品 10/10・値 10/10（100%） が一致しました。 ただしこれは「部品集合・値の対応」であって、配線のトポロジまで含めた本当の再現ではありません。 LLM に実際に回路図を生成させて比べる本命は、まだ 未計測 として分けています。
  • 解説ベンチ（回路図 → 解説の無矛盾）: 回路図から読み取れる事実（電源レール・LED やスイッチの数・ USB の有無・主要 IC など）が、プロジェクトの README の記述と矛盾しないかを照合。Tomu では 検査 7・一致 5・未確認 2・矛盾 0。「ボタン 2 個」は回路図側がタッチパッドでスイッチ記号が無いため、 矛盾ではなく 未確認 に落としています。断定はしません。

使った分だけ見えるように

これから LLM を本格的に使う段階に入るので、その前に API の使用量を実測で記録するメーターを 入れました。生成のたびに、実際に消費したトークン数が結果に残ります。推定ではなく実数です。

派手な数字はまだありません。でも、測れることを測れる形にして、正直に積み上げていきます。 続きはまたこのブログで。
